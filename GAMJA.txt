/*
 * EquipAbility Parser Design
 * 장비 능력치 파서 설계
 *
 * 이 파일은 roBrowser 장착 아이템 데이터를 기반으로,
 * 아이템 / 카드 / 인챈트 / 랜덤옵션 설명문을 수집한 뒤
 * RULE 기반 파서로 장비 능력치를 계산한다.
 *
 * 기본 원칙:
 * ID → Description → Parser → Stat Calculation
 *
 * 데이터 출처:
 * - item.ITID → DB.getItemInfo(item.ITID).identifiedDescriptionName
 * - item.slot.card1~4 → DB.getItemInfo(cardID).identifiedDescriptionName
 * - item.Options[index, value] → DB.getOptionName(index) 후 %d 치환
 *
 * 컨텍스트:
 * - item.RefiningLevel → ctx.refine
 * - item.enchantgrade  → ctx.grade
 * - opt.value          → ctx.optionValue
 *
 * 처리 흐름:
 * 1. 장착 아이템의 모든 설명문을 수집한다.
 * 2. 설명문을 줄 단위로 파서에 전달한다.
 * 3. RULE 정규식과 매칭한다.
 * 4. 매칭 성공 시 능력치를 합산한다.
 * 5. 매칭 실패 시 [ERROR] 원문을 출력한다.
 *
 * Priority:
 * 1. OPTION_MAP: random option ID based handling
 * 2. RULES: description text parser
 * 3. ERROR fallback
 *
 * Key Concept:
 * IDs define meaning, the parser defines calculation.
 */
/*
 * EquipAbility Parser Design
 * Equipment Ability Parser System
 *
 * ⚠️ Notice:
 * This parser is based on the Korean (KO) version.
 * All parsing rules and patterns are written using Korean description texts.
 *
 * This file parses equipment data from roBrowser by collecting
 * descriptions from items, cards, enchants, and random options,
 * then calculating total stats using a RULE-based parser.
 *
 * Core Principle:
 * ID → Description → Parser → Stat Calculation
 *
 * Data Sources:
 * - item.ITID → DB.getItemInfo(item.ITID).identifiedDescriptionName
 * - item.slot.card1~4 → DB.getItemInfo(cardID).identifiedDescriptionName
 * - item.Options[index, value] → DB.getOptionName(index) with %d replaced by value
 *
 * Context:
 * - item.RefiningLevel → ctx.refine
 * - item.enchantgrade  → ctx.grade
 * - opt.value          → ctx.optionValue
 *
 * Processing Flow:
 * 1. Collect all descriptions from equipped items.
 * 2. Split descriptions into individual lines.
 * 3. Match each line against RULE patterns.
 * 4. If matched, apply stat calculation.
 * 5. If not matched, output [ERROR] with the original line.
 *
 * Priority:
 * 1. OPTION_MAP: ID-based handling for random options
 * 2. RULES: description-based parsing
 * 3. ERROR: fallback for unknown patterns
 *
 * Key Concept:
 * IDs define meaning, the parser defines calculation.
 */
